Basis-Gated Memory Fusion

NANDA Memory Fusion

A stateless verification API for basis-gated agent memory decisions. It is not a long-term conversation memory store; it uses public demo data to show how a project prompt creates a basis, how source text becomes saturation, and how the fusion engine accepts or ignores reports.

Use it as a proof-of-concept and teaching harness, not production memory infrastructure. For real deployments, port or self-host the pattern inside your own trusted stack with persistence, authentication, provenance, and rate limits.

Deployment pattern example: use basis-gated fusion before or inside durable memory workflows. It can run as an external pre-write gate, inside an LLM wiki workflow, or as a local library in a trusted agent stack.

1. Assemble the task graph

A project prompt creates a node and declares the only basis dimensions that can count as evidence. Here, the calculator task needs add, subtract, multiply, and divide before it can ship.

2. Context gets saturated

The agent sees useful reports mixed with narrative text, code-shaped content, and an off-basis claim. The noisy items are present, but presence alone does not make them actionable memory.

3. Only legal reports bond

Reports that restrict onto the declared basis fuse into the task node. Natural language, code-shaped saturation, and horoscope evidence stay disconnected and cannot move the PN-Counter score.
